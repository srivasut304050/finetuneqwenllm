PEFT Library

Parameter-Efficient Fine-Tuning. Enables freezing the base model and training only 1-2% of extra weights, dramatically reducing GPU memory usage.

TRL Library

Transformer Reinforcement Learning. Provides high-level trainer classes such as SFTTrainer for streamlined fine-tuning workflows.

Accelerate Library

Handles the raw hardware allocation (multi-GPU, mixed precision FP16/BF16) behind the scenes without rewriting PyTorch loops.

2. Dataset Exploration & Structuring

To train models to follow instructions or behave as chat agents, we must format our unstructured data into standard representation structures. The modern standard is the ChatML Format.

Crucial Preprocessing Steps:
  • Ensure conversational turns are clean and structured.
  • Apply system prompts uniformly so the model maintains its persona.
  • Remove duplicates and truncate overly long documents to keep memory footprint bounded.

3. Tokenization & Attention Masks

Text cannot be read directly by neural networks. Tokenizers convert characters into integers (Token IDs), pad lists of numbers to ensure equal sizes, and construct masks to ignore filler padding.

Raw Text Input

"Fine-tuning Qwen"

Qwen Tokenizer

[BPE Vocab Lookup]

Token IDs (Input IDs)

[2784, 18223, 851, 937]

Padding: Extends short lists with specific pad tokens (e.g. [PAD]) to create uniform tensor shapes.
Attention Mask: Set to 1 for actual tokens and 0 for padding, preventing the model from attending to padding.

4. Model Loading & Quantization (QLoRA)

Quantization compresses model weights from high precision (like FP32 or FP16) to lower precision (typically 4-bit) to conserve huge amounts of VRAM. QLoRA (Quantized LoRA) uses this compressed model as a frozen base.

VRAM Footprint Comparison

For a 7 Billion parameter model (like Qwen2.5-7B):

16-bit Full Precision ~16 GB VRAM
8-bit Quantized Model ~8 GB VRAM
4-bit Quantized Model (QLoRA standard) ~5.5 GB VRAM

BitsAndBytes Config Example

python 
bnb_config = BitsAndBytesConfig(
  load_in_4bit=True,
  bnb_4bit_quant_type="nf4",
  bnb_4bit_compute_dtype=torch.bfloat16,
  bnb_4bit_use_double_quant=True
)

5. PEFT / LoRA Adapter Concept

LoRA (Low-Rank Adaptation) bypasses full weight training by injecting tiny trainable decomposition matrices in parallel to the frozen model layer parameters. Only these matrices are updated.

Input Vector (X)
→
Frozen Base Weights

Weight Matrix W₀

d x k dimensions

+
Matrix A (Down-projection)

Shape: d x r

Initialized: Random Gaussian

Matrix B (Up-projection)

Shape: r x k

Initialized: Zero (starts at 0 delta)

=
Modified Output (Y)
Y = X·W₀ + (α/r)X·AB

Rank (r)

Determines the dimensionality of the adapters. Common values are 8, 16, or 32. Lower rank uses less memory, while higher rank captures more complex behavior but increases compute size.

Alpha (α)

Scaling factor that acts as a learning rate multiplier for the adapter weights. Commonly configured to be double the value of the rank (e.g. if r = 16, then lora_alpha = 32).

Target Modules

Specifies which linear projection modules in the base model receive adapters. For Qwen, targeting q_proj, k_proj, v_proj, and o_proj yields the best results.

6. Training Execution (Supervised Fine-Tuning)

Using Hugging Face TRL, we run SFT (Supervised Fine-Tuning). SFT adjusts model probabilities to output tokens that match our training data target responses closely.

python 
from trl import SFTTrainer
from transformers import TrainingArguments

training_args = TrainingArguments(
  output_dir="./outputs",
  per_device_train_batch_size=4,
  gradient_accumulation_steps=4, # Simulates larger batch sizes
  learning_rate=2e-4,            # Typical LoRA rate
  fp16=True,                       # Use mixed-precision
  logging_steps=10,
  max_steps=200
)

trainer = SFTTrainer(
  model=model,
  train_dataset=dataset,
  peft_config=lora_config,
  dataset_text_field="text",
  args=training_args
)
trainer.train()

7. Evaluation, Merging & Deployment

After training, we are left with a folder of tiny weight adjustments (the LoRA adapter, ~20MB). To deploy this model for local execution, we merge these adapters back into the multi-gigabyte base model weights.

1. Merge Weight Matrices

We load the original base model alongside our trained adapter weights and perform the math: W_new = W₀ + (α/r)·AB. We save the resulting model as a complete, standalone LLM.

2. Export to GGUF format

Using tools like llama.cpp, we convert the merged model into GGUF format and quantize it to 4-bit (Q4_K_M) or 8-bit. This optimizes it to run locally on CPU/GPU.
