The 8-Step Fine-Tuning Pipeline

Fine-tuning takes a pre-trained base model and customizes it to answer specific prompts. Here is exactly what happens at every step, explained simply.

Step 1

Setting Up the Virtual Sandbox

The Analogy
"Imagine baking a cake. If you bake in a cluttered kitchen with random spices everywhere, your cake might taste weird. A virtual environment is a brand-new, clean kitchen dedicated exclusively to baking this one cake."
  • What happens: Python isolates a folder (`.venv`) so the library versions we install for this project don't conflict with other projects on your computer.
  • Why we do it: Machine learning packages (like PyTorch) are sensitive to version differences. Separation ensures stable execution.
Local Setup Command:
# 1. Create the clean virtual sandbox
python -m venv .venv

# 2. Step into the sandbox (Windows)
.venv\Scripts\activate

# 3. Install core LLM libraries
pip install -r requirements.txt
Step 2

Preparing the Dataset (The Textbooks)

The Analogy
"If you want to train a student to pass a coding exam, you need to compile a study guide full of clean, correct code questions and answers. The dataset is this textbook."
  • What happens: We gather data and format it into conversations. We use **ChatML Format** to explicitly label who is speaking.
  • Why we do it: Pre-trained models only know how to complete text. ChatML tags teach the model when it is the user's turn to speak and when it is the assistant's turn to answer.
Visual ChatML Box Structure:
JSON Representation (data/raw/sample.json)
System "You are an expert coder."
User "What is Python?"
Assistant "Python is an interpreted programming language..."
Step 3

Tokenization (Translating Words to Numbers)

The Analogy
"An LLM doesn't speak English; it speaks math. Think of tokenization as translating characters into discrete puzzle pieces, and assigning each piece a unique catalog code number."
  • What happens: Text is split into sub-word units (Tokens) and mapped to integer Token IDs. An **Attention Mask** (1s and 0s) is created to tell the model to ignore empty space.
  • Why we do it: Neural networks perform math equations, so words must be converted to numerical matrices before entering the model.

Step 4

Model Loading & 4-bit Quantization

The Analogy
"Imagine a massive encyclopedia library. If you want to carry it on a bus, you can't fit the physical books. But if you compress the text to microfilm, you can carry it in your pocket."
  • What happens: Model weights are converted from 16-bit float formats (which take up massive memory) to compressed 4-bit formats (using `bitsandbytes` NF4).
  • Why we do it: Reduces a 7B model from 15 GB down to only 5.5 GB. This allows us to load the model on a standard GPU (like the free Google Colab T4).

Quantization Loss:

Compressing values to 4-bit does reduce model accuracy slightly. However, QLoRA fixes this by using a high-precision compute format (BFloat16) during training calculations.

bitsandbytes_config = BitsAndBytesConfig(
  load_in_4bit=True,
  bnb_4bit_quant_type="nf4"
)
Step 5

PEFT & LoRA (The Sticky Note Method)

The Analogy
"Instead of rewriting a whole encyclopedia by hand to update it (which takes weeks), you stick small post-it notes on the pages with corrections. The original book is locked (frozen), and you only write updates on the sticky notes."
  • What happens: We freeze all base model weights. We introduce thin, trainable adapter weight layers (Matrices A & B) alongside the base model weights.
  • Why we do it: We only update 1% of the model's total weights. This reduces training memory and compute requirements by up to 90%.

Step 6

Training Loop (The Study Session)

The Analogy
"The model reads a question, guesses the answer, checks the answer sheet, sees how wrong it was (the Loss), and slightly adjusts its sticky notes. It repeats this thousands of times until it gets the answers right."
  • What happens: SFTTrainer feeds token batches, calculates cross-entropy loss against target tokens, and updates adapter weights using backpropagation.
  • Why we do it: This updates the parameters of the LoRA matrices so the model learns the styling, tone, and specific rules of the dataset.
Hyperparameters Demystified:
Learning Rate: The step size at each correction. Too high ruins the model; too low takes forever (Standard: 2e-4).
Gradient Accumulation: Simulates larger training batches on low VRAM GPUs by accumulating gradients over multiple smaller steps.
Step 7

Merging the Adapters (Glueing the Notes)

The Analogy
"To publish the updated encyclopedia, you print a new edition where the sticky notes are merged directly into the pages, resulting in a single clean book."
  • What happens: We combine the original base model weights with the trained LoRA adapter weights, saving it as a standalone model folder.
  • Why we do it: Merging removes inference latency and makes the model compatible with standard deployment engines (like Ollama).

Latency Free

Once merged, the model runs at full native speed with no extra calculation steps.

Step 8

Local Deployment (Running Offline)

The Analogy
"Now that you have your new book, you download a reader app (like Kindle) onto your laptop so you can read it offline whenever you want, without needing the internet."
  • What happens: We convert the merged model to GGUF format and load it locally via Ollama and Open WebUI.
  • Why we do it: Allows you to query your fine-tuned model instantly, locally, for free, with complete data privacy.
Local Run Command:
# Start model in local terminal
ollama run my-qwen-model

How local VS Code connects to Google Colab

You do not have an NVIDIA GPU locally, but Google Colab provides one for free. Here is the operational loop showing how your files get to Colab and back.

1

Write Code in VS Code

Write your files, templates, and yaml configurations in your local environment.

2

Push Code to GitHub

Commit your local changes and push them to a private/public GitHub repository.

3

Clone into Google Colab

Open Google Colab, connect to a GPU, and clone your repository using Git.

4

Run Training on Colab GPU

Execute the training script. Colab fetches the model, quantizes it, and saves adapters.

5

Push Adapters to Hugging Face

Upload the small LoRA adapters directly to Hugging Face for easy retrieval locally.

Colab GPU Commands

Paste these commands directly into a Google Colab notebook cell to link your workspace and execute training:

# 1. Clone your VS Code repository
!git clone https://github.com/username/qwen-finetuning.git

# 2. Enter workspace directory
%cd qwen-finetuning

# 3. Install packages on GPU machine
!pip install -r requirements.txt

# 4. Trigger training run
!python src/train.py
Pro Tip: Adapters are small (~20MB) and quick to transfer, while base models are huge (~15GB). Only sync the adapters back to your local PC.
